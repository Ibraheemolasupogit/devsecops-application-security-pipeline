# Methodology

The principal method is STRIDE:

- Spoofing
- Tampering
- Repudiation
- Information disclosure
- Denial of service
- Elevation of privilege

The model also incorporates:

- OWASP API Security Top 10.
- Abuse cases.
- Multi-step attack paths.
- Identity and authorisation risks.
- Cloud-native risks.
- Software supply-chain risks.
- Audit and logging risks.
- Data confidentiality risks.
- Workflow integrity risks.

Threats are created where they materially affect the current API or a credible future production deployment. The register avoids one-threat-per-category padding.
